Plano: Conexão Frontend ↔ BFF                                                                                            
     Contexto                                                                                                                                                                                                                                       
     O frontend (React + Redux Toolkit) e o BFF (Express + MongoDB) estão implementados parcialmente mas desconectados. O
     frontend faz chamadas HTTP para 5 endpoints que o BFF não expõe no formato esperado. O objetivo é implementar o
     contrato de API completo entre os dois, seguindo a Clean Architecture já estabelecida em ambos os packages.

     ---
     Mapeamento do Contrato de API

     O Nginx do frontend faz strip de /api/ antes de encaminhar ao BFF. Portanto:

     ┌─────────────────────────────────┬─────────────────────────────┬────────────┐
     │         Frontend chama          │       BFF deve expor        │   Status   │
     ├─────────────────────────────────┼─────────────────────────────┼────────────┤
     │ POST /api/diagrams/upload       │ POST /diagrams/upload       │ ❌ Ausente │
     ├─────────────────────────────────┼─────────────────────────────┼────────────┤
     │ GET /api/analysis/:id           │ GET /analysis/:id           │ ✅ Existe  │
     ├─────────────────────────────────┼─────────────────────────────┼────────────┤
     │ GET /api/sessions               │ GET /sessions               │ ❌ Ausente │
     ├─────────────────────────────────┼─────────────────────────────┼────────────┤
     │ POST /api/sessions/:id/messages │ POST /sessions/:id/messages │ ❌ Ausente │
     ├─────────────────────────────────┼─────────────────────────────┼────────────┤
     │ GET /api/sessions/:id/messages  │ GET /sessions/:id/messages  │ ❌ Ausente │
     └─────────────────────────────────┴─────────────────────────────┴────────────┘

     Etapa 7 — Frontend: Camada de API Client

     Criar packages/frontend/src/infrastructure/api/bffClient.ts com funções tipadas:

     export const bffClient = {
       uploadDiagram(file: File, sessionId: string): Promise<{ diagramId: string; analysisId: string }>,
       pollAnalysis(id: string, opts?: { intervalMs?: number; maxAttempts?: number }): Promise<AnalysisResult>,
       getSessions(): Promise<SessionRecord[]>,
       createSession(name: string): Promise<SessionRecord>,
       getMessages(sessionId: string): Promise<ChatMessage[]>,
       sendMessage(sessionId: string, content: string, attachments?: MessageAttachment[]): Promise<ChatMessage>,
     };

     pollAnalysis implementa polling com setInterval/setTimeout:
     - Intervalo padrão: 2000ms
     - Timeout: 30 tentativas (~60s)
     - Resolve quando status === 'completed' | 'failed'
